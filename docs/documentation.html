Yaex is a JavaScript library for modern browsers with a largely jQuery/Zepto compatible API . If you use jQuery/Zepto, you already know how to use Yaex.

While 100% jQuery/Zepto coverage is not a design goal, the APIs provided match their jQuery/Zepto counterparts . The goal is to have a small modular library that downloads and executes fast , with a familiar and versatile API , so you can concentrate on getting stuff done .

Yaex is open source software and is released under the developer and business-friendly MIT license.

Browser Support

Primary (100% support)

  • Safari 6+ (Mac)
  • Chrome 30+ (Windows, Mac, Android, iOS, Linux, Chrome OS)
  • Firefox 24+ (Windows, Mac, Android, Linux, Firefox OS)
  • iOS 5+ Safari
  • Android 2.3+ Browser
  • Internet Explorer 10+ (Windows, Windows Phone)

Secondary targets (fully or mostly supported)

  • iOS 3+ Safari
  • Chrome <30
  • Firefox 4+
  • Safari <6
  • Android Browser 2.2
  • Opera 10+
  • webOS 1.4.5+ Browser
  • BlackBerry Tablet OS 1.0.7+ Browser
  • Amazon Silk 1.0+
  • Other WebKit-based browsers/runtimes

Note that some optional features of Yaex specifically target mobile browsers; as the original project goal was to specifically provide a leaner alternative to jQuery/Zepto for the mobile web.

Yaex is a good choice for browser extensions (for Safari, Chrome and Firefox) and to develop HTML-based views within native app frameworks, such as PhoneGap.

In short, Yaex is expected to work in every modern browser and browser-like environment. Yaex doesn't support old Internet Explorer versions (<10).

Manually Building

yyaex_core_src.js and yaex_core_min.js provided above can be used as-is. However, for best efficiency and customizability, run the build system that comes with Yaex's source code. It allows you to select modules, run tests, use UglifyJS to minify your custom build, and gives you an estimate on the compression that is achievable when yaex_core_min.js is served gzipped.

Refer to the README for instructions on how to build Yaex, including running the tests and contributing patches.
